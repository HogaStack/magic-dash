# `magic-dash-pro`项目创建

`magic-dash-pro`是适用于多页面、持续扩展的管理型应用模板，内置用户登录、用户管理、部门管理、角色权限、数据库模型和鉴权能力，适合构建需要复杂用户与部门关系的业务系统。创建时可以选择`Flask`或`FastAPI`后端变体。

## 创建项目

```bash
magic-dash create --name magic-dash-pro
```

命令会先提示选择后端类型：

- `Flask`：默认后端，基于`flask-login`
- `FastAPI`：可选后端，基于`fastapi-login`

🚀 后端选择建议：

- 当项目需要更强的并发服务能力、原生异步支持、`WebSocket`等能力时，推荐选择`FastAPI`后端。
- 当项目更偏轻量化，且希望代码编写和后续维护更简单时，推荐选择`Flask`后端。

随后会提示输入项目名称。直接回车时，默认生成`magic-dash-pro`目录。

指定项目生成父目录：

```bash
magic-dash create --name magic-dash-pro --path ./workspace
```

直接指定后端类型：

```bash
magic-dash create --name magic-dash-pro --backend flask --path ./workspace
magic-dash create --name magic-dash-pro --backend fastapi --path ./workspace
```

> `magic-dash-pro-fastapi`是内部后端变体，不能作为顶层模板通过`--name`直接创建。

## 安装依赖、初始化并启动

```bash
cd magic-dash-pro
pip install -r requirements.txt
python -m magic_init
python app.py
```

默认访问地址：

```text
http://127.0.0.1:8050
```

若创建时选择`FastAPI`后端，部署时推荐使用：

```bash
uvicorn app:app.server --host 0.0.0.0 --port 8050 --workers 4
```

初始化成功后，默认管理员账号为：

```text
用户名：admin
初始密码：admin123
```

生产环境请在首次登录后尽快修改默认密码，并替换应用密钥和数据库连接信息。

## 初始项目结构

```text
magic-dash-pro/
├── app.py
├── server.py
├── magic_init.py
├── requirements.txt
├── assets/
├── callbacks/
├── components/
├── configs/
├── models/
├── utils/
└── views/
```

关键目录说明：

| 路径 | 作用 |
| --- | --- |
| `configs/` | 应用、布局、路由、鉴权、数据库配置 |
| `models/` | 用户、部门、登录日志等数据库模型 |
| `views/` | 登录页、核心页面、状态页 |
| `callbacks/` | 登录、页面交互和核心页面回调 |
| `components/` | 侧边菜单、页面容器、用户管理等自定义功能组件 |
| `utils/` | 加密、缓存清理等工具函数 |
