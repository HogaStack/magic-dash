# `simple-tool`二次开发指南

`simple-tool`的核心开发入口是`app.py`。它把页面布局、输入组件、提交按钮、结果容器和回调函数放在同一个文件中，便于快速改造。

## 修改页面信息

常见改动集中在以下位置：

- `dash.Dash(..., title="A Basic Tool")`：修改浏览器标签页标题
- 页面顶部`fac.AntdText("基础工具标题")`：修改工具标题
- 页面说明文案：替换为工具的输入要求、输出说明或注意事项
- 主面板样式：调整宽度、内边距、阴影、背景色等视觉参数

## 调整输入区

模板默认生成 4 个输入项：

```python
id=f"demo-input{i}"
```

开发时通常需要做三件事：

1. 将示例输入项替换为真实业务字段。
2. 为每个输入组件设置稳定、语义化的`id`。
3. 在回调函数的`State(...)`中同步更新这些`id`。

需要更丰富的输入类型时，可以替换为`fac.AntdSelect`、`fac.AntdInputNumber`、`fac.AntdDatePicker`、`fac.AntdUpload`等组件。

## 编写计算逻辑

模板中的`update_result()`是主要业务逻辑入口：

```python
@app.callback(...)
def update_result(nClicks, value1, value2, value3, value4):
    ...
```

建议将真实计算逻辑拆到独立函数中，再在回调里负责参数校验、调用计算函数和渲染结果。这样后续增加测试或迁移到多页面模板时会更轻松。

## 输出结果

结果区域由`result-container`承载。回调函数只需要返回合法的计算结果展示组件内容即可。

## 项目变大后的拆分建议

更复杂的多页面应用请更换成基于[`magic-dash`](../magic-dash/项目创建.md)或[`magic-dash-pro`](../magic-dash-pro/项目创建.md)模板构建。
