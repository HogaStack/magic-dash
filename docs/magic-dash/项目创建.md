# `magic-dash`项目创建

`magic-dash`是适用于多页面、持续扩展的应用模板，内置侧边菜单、核心页面容器、路由配置、状态页、页面搜索和多标签页布局能力，适合构建需要长期迭代的多页面数据应用。

## 创建项目

```bash
magic-dash create --name magic-dash
```

命令会先提示选择后端类型，再提示输入项目名称。直接回车时，默认生成`magic-dash`目录。

后端类型可选择：

- `Flask`：默认后端
- `FastAPI`：生成后会将`dash.Dash()`实例切换为`backend="fastapi"`，在`requirements.txt`中补充`dash[fastapi]`、`fastapi`、`uvicorn`等依赖，并将浏览器版本检查从`Flask before_request`改写为`FastAPI middleware`

指定项目生成父目录：

```bash
magic-dash create --name magic-dash --path ./workspace
```

直接指定后端类型：

```bash
magic-dash create --name magic-dash --backend flask --path ./workspace
magic-dash create --name magic-dash --backend fastapi --path ./workspace
```

等价的简写形式：

```bash
magic-dash create -n magic-dash -b flask -p ./workspace
magic-dash create -n magic-dash -b fastapi -p ./workspace
```

## 安装依赖并启动

```bash
cd magic-dash
pip install -r requirements.txt
python app.py
```

默认访问地址：

```text
http://127.0.0.1:8050
```

## 初始项目结构

```text
magic-dash/
├── app.py
├── server.py
├── requirements.txt
├── assets/
├── callbacks/
├── components/
├── configs/
├── utils/
└── views/
```

关键目录说明：

| 路径 | 作用 |
| --- | --- |
| `configs/` | 应用标题、布局模式、路由、菜单等配置 |
| `views/core_pages/` | 核心页面渲染模块 |
| `views/status_pages/` | 404、500 等状态页 |
| `callbacks/` | 页面回调和交互逻辑 |
| `components/` | 自定义功能组件 |
| `assets/` | CSS、JS、图片、图标等静态资源 |
