# `magic-dash`二次开发指南

`magic-dash`模板适合按“页面渲染 + 路由配置 + 回调逻辑”的方式组织多页面、持续扩展的应用。开发时建议先确认页面类型，再修改对应配置。

## 新增普通核心页面

1. 在`views/core_pages/`下新增页面文件，例如：

```text
views/core_pages/report.py
```

2. 编写页面渲染函数：

```python
def render():
    return ...
```

3. 在`configs/router_config.py`中加入菜单项：

```python
{
    "component": "Item",
    "props": {
        "title": "报表页",
        "key": "/core/report",
        "icon": "antd-bar-chart",
        "href": "/core/report",
    },
}
```

4. 在`valid_pathnames`中登记`pathname`：

```python
"/core/report": "报表页"
```

5. 在`components/page_content.py`中导入新页面模块，并在`render()`函数中补充分发逻辑：

```python
from views.core_pages import report

...

elif pathname == "/core/report":
    page_content = report.render()
```

6. 如需回调，在`callbacks/core_pages_c/`下新增对应回调文件。可以参考`page1.py`的写法，在页面模块中导入对应回调模块，或在包初始化逻辑中集中导入，确保回调注册生效。

## 新增独立页面

独立页面适合详情页、分享页、全屏页等不需要侧边菜单框架的页面。

处理方式：

- 在`views/core_pages/`下新增渲染模块。
- 在`valid_pathnames`中加入页面路径。
- 在`independent_core_pathnames`中加入同一个路径。
- 在`views/core_pages/__init__.py`的独立渲染判断中导入并返回该页面的`render()`结果。
- 如果需要入口页，可以在`core_side_menu`中增加跳转入口。

## 新增通配页面

通配页面适合详情页这类路径中包含动态参数的页面，例如`/core/report/detail/123`。

处理方式：

1. 在`wildcard_patterns`中定义正则。
2. 将正则对象加入`valid_pathnames`。
3. 如页面需要独立渲染，将正则对象加入`independent_core_pathnames`，并在`views/core_pages/__init__.py`中补充命中该正则后的页面渲染分支。
4. 在页面渲染函数中从`URL`中解析动态参数。

当前模板内置流程主要演示“独立通配页面”。如果希望新增嵌入核心布局的普通通配页面，还需要同步调整`callbacks/core_pages_c/__init__.py`中的路径合法性判断和多标签页标题生成逻辑，并在`components/page_content.py`中补充对应渲染分发。

## 调整菜单和首页

- 修改首页地址：调整`RouterConfig.index_pathname`。
- 修改侧边菜单：调整`RouterConfig.core_side_menu`。
- 控制子菜单默认展开：调整`RouterConfig.side_menu_open_keys`。
- 切换多标签页：将`LayoutConfig.core_layout_type`改为`"tabs"`。

## 静态资源与全局样式

静态资源放在`assets/`目录下：

- `assets/css/`：全局样式
- `assets/js/`：客户端脚本
- `assets/imgs/`：图片资源
- `assets/favicon.ico`：站点图标

[Dash](https://github.com/plotly/dash)会自动加载`assets/`下的静态资源。新增资源后，使用`/assets/...`路径引用。

## 何时升级到`magic-dash-pro`

当项目需要登录、角色权限、用户管理、登录日志或数据库模型时，建议基于[`magic-dash-pro`](../magic-dash-pro/项目创建.md)重新创建项目，或将现有页面迁移到`pro`模板。
